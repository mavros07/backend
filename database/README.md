# Database files

Same layout as **Stay Eazi Booking Engine** (`app.stayeazi.com/database`): **`stayeazi_eaziapp_db.sql`** + **`manual_mysql_patches.sql`** + **`README.md`**. Here: **Laravel migrations are canonical**; SQL files support review, backup, and hosts **without** Artisan.

| Item | Purpose |
|------|--------|
| **`migrations/`** | **Canonical** schema — run `php artisan migrate` on each environment (after backup). Only `*.php` files belong here. |
| **`myauto_torque_db.sql`** | **MySQL/MariaDB baseline — structure only** (`CREATE TABLE`). Same role as Stay Eazi’s main dump header: edit when migrations change. |
| **`myauto_torque_db_data.sql`** | **Public site content + settings** — `INSERT`s for `roles` (admin/user), `site_settings`, and **`cms_pages`** (home, about, faq, contact) from `database/seed-data/*.html`. Regenerate with `php artisan db:generate-mysql-baseline-data`. |
| **`myauto_torque_db_full.sql`** | **Single import file** — concatenation of structure + data (generated by the same Artisan command). Use for phpMyAdmin “one shot” after creating an empty database. **Does not** include user accounts or listings (those use Laravel register/login against the same DB). |
| **`manual_mysql_patches.sql`** | **Fallback only** — hosts that **cannot** run Artisan (one-off `ALTER TABLE`, indexes, charset fixes). Prefer `php artisan migrate` everywhere you can. |

Prefer adding schema **only** via **`migrations/`**. Use the SQL files when you need a portable MySQL script or a host without CLI.

---

## Production / staging workflow

1. After backup:

   ```bash
   php artisan migrate:status
   php artisan migrate --force
   ```

2. Seed baseline content when appropriate (CMS, site settings, demo data), **or** import `myauto_torque_db_full.sql` / run:

   ```bash
   php artisan db:generate-mysql-baseline-data   # refreshes *_data.sql and *_full.sql from seed HTML
   ```

   ```bash
   php artisan db:seed --force
   ```

3. Optional: export a **full** dump (structure + data) from phpMyAdmin or:

   ```bash
   mysqldump -u USER -p myauto_torque > database/myauto_torque_db_full_backup.sql
   ```

   Keep large dumps **out of git** if they contain real customer data; store in secure backup storage.

---

## If you import `myauto_torque_db.sql` manually

1. Create the database (utf8mb4) and import the file.
2. Laravel expects a `migrations` table. After import, either:
   - run `php artisan migrate` (should no-op if schema matches), or  
   - insert rows into `migrations` for each `database/migrations/*.php` already reflected in the DB (maintenance window — only if you know each change is applied).

3. Run `php artisan db:seed --force` if you need default CMS/site settings.

## Environment

Default database name in `.env.example`: **`myauto_torque`**. Adjust `USE \`myauto_torque\`` in SQL files if your server uses another name.

## First admin (temporary `/bootstrap-admin`)

Same idea as Prestige `CREATE_ADMIN_TEMP.php`: enable **`ADMIN_BOOTSTRAP_ENABLED=true`** in `.env`, open **`/bootstrap-admin`**, create the first admin (Spatie role `admin`), then set **`ADMIN_BOOTSTRAP_ENABLED=false`** and remove the `if (config('app.admin_bootstrap_enabled')) { ... }` block from `routes/web.php` so the URL is gone.

Routes are **not** registered when the flag is false (they will not appear in `php artisan route:list` until enabled).
